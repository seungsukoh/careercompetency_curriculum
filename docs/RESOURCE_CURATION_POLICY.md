# 자료 큐레이션 기준

## 기본 방향

초기 파일럿은 한국어로 접근 가능한 자료를 우선 사용한다. 다만 직무와 직접 맞는 공식 문서, 기업 예제, 글로벌 공개강의는 영어 자료도 사용할 수 있다. 학생은 번역 도구로 영어 자료를 볼 수 있으므로, 앱은 모든 자료에 `한국어` 또는 `영어`를 명확히 표시한다.

자료는 단순 링크 목록이 아니라 학생 상황에 맞는 커리큘럼 구성 단위로 관리한다. 따라서 모든 자료에는 난이도, 예상 소요시간, 선행조건, 연결 역량, 산출물을 붙인다.

강좌 전체 링크만 넣지 않는다. 가능한 경우 학생이 실제로 볼 단원, 문서 섹션, 예제 범위를 `recommendedSections`에 기록한다. 검색 페이지나 플랫폼 후보는 `candidate`와 `broad`로 두고, 운영자가 강좌 1개와 볼 단원까지 좁힌 뒤 `reviewed`로 승격한다.

`reviewed` 이상으로 승격하려면 학생이 누를 최종 URL을 본문 GET 기준으로 열어 제목, 본문 키워드, 목차 또는 파일명을 확인한다. 리다이렉트가 홈으로 떨어지거나 403/404/로그인 차단이 발생한 자료는 학생용 추천에서 제외하고 `candidate` 또는 `reviewNeeded`로 둔다.

`core: true` 자료 중 `reviewed` 또는 `verified` 상태인 자료는 Starter Pack으로 취급한다. Starter Pack은 선택 자료가 아니라 해당 직무 커리큘럼을 시작하기 전에 먼저 완료해야 할 핵심 역량 자료로 안내한다. `candidate`와 `reviewNeeded` 자료는 핵심 후보여도 Starter Pack에 노출하지 않는다.

## 우선 출처

1. K-MOOC, KOCW, 국내 대학 공개강의
2. NCS, STEP, HRD-Net, 공공기관/협회 자료
3. 기업/기관의 한국어 공식 기술자료
4. 한국어 YouTube 강의와 플레이리스트
5. Boostcourse, 디지털배움터, GSEEK, 엘리스 등 한국어 실습형 공개 과정
6. KIRD, KOSHA 등 연구개발·안전·공정 관련 공공 교육자료
7. 검수된 개인/커뮤니티 자료

## 한국어 후보 출처 풀

한국어 자료가 부족한 트랙은 아래 출처를 먼저 확인한다. 단, 플랫폼 메인 링크는 최종 자료가 아니라 후보 풀이다. 실제 반영은 직무 과제와 맞는 강좌 1개, 볼 단원 1-2개, 산출물을 정한 뒤 진행한다.

| 출처 | 우선 확인 주제 | 반영 조건 |
|---|---|---|
| K-MOOC | 전공기초, 공학 일반, AI·데이터 | 강좌 목차에서 직무 과제와 직접 맞는 단원이 확인될 때 |
| KOCW | 대학 전공강의, 회로/기계/화공/반도체 | 검색 결과가 아니라 특정 강좌와 단원까지 좁혔을 때 |
| STEP | 산업기술 실습, 직무교육 | 실습 산출물 또는 평가 과제가 있는 과정일 때 |
| HRD-Net | 직업훈련, 자격·도구 보완 | NCS 능력단위와 교과목이 지원 직무와 맞을 때 |
| NCS | 직무능력, 수행준거, 학습모듈 | 교육자료보다 직무 행동 기준 보강용으로 사용 |
| Boostcourse | Python, 데이터, AI, SW 실습 | 실습 코드나 프로젝트 결과물이 남는 과정일 때 |
| 디지털배움터/GSEEK | AI·IT 입문, 취업기초, 문서화 | 전공 심화 전 기초 보완 목적일 때 |
| 엘리스 | Python, AI, 데이터 실습 | 무료 공개 범위와 실습 산출물을 확인한 뒤 후보화 |
| KIRD | R&D, 기술기획, 연구윤리, 안전 | 공개 수강 가능하고 R&D 직무 산출물과 연결될 때 |
| KOSHA | 공정안전, PSM, MSDS, 위험성평가 | 화학·반도체·안전 과제의 체크리스트로 연결될 때 |

## YouTube 자료 선별 기준

YouTube는 조회수와 댓글이 많은 자료를 우선 후보로 삼는다. 단, 조회수와 댓글은 인기 신호일 뿐 최종 품질 판단 기준은 아니다.

우선 검토할 영상:

- 조회수가 같은 주제의 다른 영상보다 높은 영상
- 댓글에서 질문과 답변이 실제로 오가는 영상
- 좋아요, 구독자, 플레이리스트 구성 등 참여 신호가 좋은 채널
- 대학, 공식 기관, 기업, 현업 전문가 채널
- 영상 길이와 학습 목표가 명확한 강의 또는 시리즈
- 최근 기술 변화가 중요한 주제는 업로드 시점이 너무 오래되지 않은 영상

주의할 영상:

- 조회수는 높지만 자극적인 제목 위주인 영상
- 댓글이 많아도 학습 질문보다 홍보성 댓글이 대부분인 영상
- 특정 유료 강의, 제품, 부트캠프 홍보가 주목적인 영상
- 실습 결과나 산출물로 이어지지 않는 단순 소개 영상
- 직무역량과 연결하기 어려운 흥미 위주 영상

## 랭킹 신호

자료 후보는 다음 신호를 함께 본다.

| 신호 | 용도 |
|---|---|
| 공식성 | 대학, 공공기관, 기업 공식 자료인지 판단 |
| 조회수 | 같은 주제 안에서 많이 소비된 자료인지 판단 |
| 댓글 품질 | 학습자가 실제로 도움을 받거나 질문하는지 판단 |
| 최신성 | 기술, 도구, 표준이 낡지 않았는지 판단 |
| 난이도 적합성 | 공학 전공 취준생이 현재 역량과 준비 기간 안에서 접근 가능한지 판단 |
| 선행조건 | 먼저 알아야 할 개념이 명확한지 판단 |
| 산출물 연결성 | 보고서, 실습 결과, 체크리스트, 코드 등으로 이어지는지 판단 |
| 시간 대비 효율 | 짧은 파일럿 커리큘럼에 넣을 만큼 밀도가 있는지 판단 |

## 추천 메타데이터

자료 데이터에는 최소한 다음 필드를 둔다.

```js
{
  id: "resource-id",
  title: "자료 제목",
  url: "https://example.com",
  type: "YouTube",
  provider: "채널 또는 기관명",
  language: "ko",
  difficulty: "입문",
  sequenceLevel: 1,
  estimatedMinutes: 45,
  practiceMinutes: 60,
  prerequisites: [],
  recommendedSections: ["볼 단원 또는 문서 섹션"],
  tracks: ["electronics-pcb"],
  skills: ["회로이론"],
  expectedOutput: "회로 개념 요약 노트",
  engagement: {
    views: null,
    comments: null,
    likes: null,
    checkedAt: "2026-06-25"
  },
  qualityStatus: "candidate"
}
```

## 커리큘럼 구성 원칙

- 단기 커리큘럼은 조회수와 댓글 신호가 좋은 입문/기초 자료를 우선 사용한다.
- 장기 커리큘럼은 공식 자료, 공개강의, 실습 자료를 섞어 선후관계를 만든다.
- 영상만 연속으로 배치하지 않고, 각 단계마다 산출물을 남기게 한다.
- 같은 주제에 자료가 여러 개 있으면 한국어 접근성, 시간 대비 효율, 댓글 품질, 공식성을 함께 비교한다.

## 운영 부담 최소화 원칙

운영자가 매주 많은 자료를 직접 찾아야 하는 구조는 피한다. 초기에는 자료 수를 제한하고, 오래 쓸 수 있는 기본 자료 묶음을 만든다.

권장 방식:

- 세부 직무별 핵심 자료 묶음을 작게 유지한다.
- 최신성이 덜 민감한 전공 기초, 직무 이해, 공식 공개강의를 우선 사용한다.
- 빠르게 낡는 도구 사용법이나 채용 트렌드 자료는 `reviewNeeded` 상태로 둔다.
- 자료별 `lastVerifiedAt`과 `nextReviewAt`을 둔다.
- 학생이 끊긴 링크나 부정확한 자료를 신고할 수 있게 한다.
- 조회수, 댓글 수, 링크 생존 여부 같은 기계적 확인은 나중에 자동화한다.
- 최종 추천 여부, 난이도, 선후관계, 산출물 판단은 사람 검수로 유지한다.

초기 검수 주기는 매주가 아니라 월 1회 또는 학기별 1회로 시작한다. 사용자가 늘어나기 전까지는 자동 크롤링보다 작고 안정적인 자료 묶음이 더 안전하다.
