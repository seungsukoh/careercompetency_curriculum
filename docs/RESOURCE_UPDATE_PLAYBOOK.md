# 자료 정기 업데이트 운영안

## 운영 주기

자료와 워드클라우드는 월 1회 또는 학기 시작 전에 함께 갱신한다.

## 작업 순서

1. `data/resource_candidates.csv`에 이번 검수 후보를 기록한다.
2. 후보는 지정 출처에서만 찾는다: KOCW, K-MOOC, STEP, MIT OCW, NCS, 공식 문서, 기업 기술자료.
3. 후보를 반드시 앱의 4주 과제 중 하나에 연결한다.
4. 강의 전체가 아니라 학생이 볼 단원, 문서 섹션, 실습 범위를 기록한다.
5. 산출물이 앱의 과제 제출물과 맞는지 확인한다.
6. 검수상태를 `candidate`, `reviewed`, `verified`, `rejected` 중 하나로 바꾼다.
7. `reviewed` 이상만 앱 데이터에 반영한다.
8. `matlab -batch "addpath(pwd); generateWordClouds"`로 워드클라우드 PNG를 재생성한다.
9. `npm.cmd run build`로 정적 배포물을 갱신한다.

## 후보 검수 기준

| 항목 | 기준 |
|---|---|
| 과제 연결 | 특정 직무 트랙과 주차 과제에 직접 연결되어야 한다. |
| 학생 행동 | 시청/읽기 후 만들 산출물이 분명해야 한다. |
| 범위 | 강의 전체가 아니라 볼 단원 또는 섹션이 지정되어야 한다. |
| 수준 | 2학년은 입문, 3학년은 기초실습, 4학년은 포트폴리오/면접 설명에 맞춘다. |
| 출처 | 공개자료, 공식문서, 대학 OCW, 기업 공식 기술자료를 우선한다. |
| 유지보수 | 링크 확인일과 다음 검토일을 기록한다. |

## 워드클라우드 갱신 기준

워드클라우드는 실시간 검색 결과가 아니라 앱의 검수된 직무/진단/과제/자료 데이터에서 반복되는 역량을 반영한다.

반복해서 등장하는 역량은 더 크게 보이게 한다. 새 자료나 과제가 추가되면 `generateWordClouds.m`의 단어와 가중치를 함께 수정한다.
