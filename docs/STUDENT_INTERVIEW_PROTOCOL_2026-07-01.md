# 학생 인터뷰 프로토콜 - 2026-07-01

## 목적

이 프로토콜은 자동 QA가 확인하지 못하는 학생 관점의 이해도를 검증하기 위한 30분 인터뷰 절차다. 핵심은 추천 교육의 개수를 늘리는 것이 아니라, 학생이 아래 세 가지를 구분하고 납득하는지 확인하는 것이다.

- 전공: 학생의 출발 배경
- 상세직무내용: 실제 채용과 업무에서 수행할 일
- 직무 관련 교육: 그 직무를 준비하기 위해 먼저 보완할 학습 자료

특히 다음 질문에 답을 얻는다.

1. 학생이 자기 전공과 추천 직무의 관계를 `전공 직결`, `전공 확장`, `도전 직무` 중 하나로 이해하는가?
2. 상세직무 카드가 막연한 산업 소개가 아니라 실제 업무와 산출물을 설명한다고 느끼는가?
3. 첫 번째 추천 교육이 왜 해당 직무에 도움이 되는지 학생이 자기 말로 설명할 수 있는가?
4. 1-2주차 교육 추천에 중복, 선후관계 역전, 너무 쉬운 자료 반복처럼 보이는 문제가 남아 있는가?
5. `큰 공백 없음` 같은 문구가 역량을 모두 체크한 학생에게 과하게 낙관적으로 보이지 않는가?

## 대상자 구성

최소 5명으로 시작한다. 가능하면 아래 조합을 포함한다.

| 구분 | 권장 대상 |
|---|---|
| 전공 직결 | 기계/전기/전자/화학 중 자기 전공과 가까운 직무를 고르는 학생 |
| 전공 확장 | 컴퓨터공학, 산업공학, 신소재/재료 등 인접 직무로 확장하는 학생 |
| 도전 직무 | 전공과 거리가 있는 AI, 자율주행, 데이터센터, 제조 DX 직무를 고르는 학생 |
| 단기 준비 | 면접 또는 과제 제출까지 2주 안팎만 남은 학생 |
| 기초 보완 | 관심 직무는 있으나 역량 체크에서 미체크 항목이 많은 학생 |

## 진행 전 준비

- 인터뷰 시간: 30분
- 진행 방식: 화면 공유 또는 옆자리 관찰
- 기록자: 1명
- 테스트 URL: 최신 배포 URL 또는 로컬 QA 통과 버전
- 금지 사항: 진행자가 먼저 기능을 설명하지 않는다. 학생이 화면의 용어와 추천 이유를 어떻게 해석하는지 먼저 듣는다.

## 사전 질문

시작 전에 3분 안에 확인한다.

| 질문 | 기록 항목 |
|---|---|
| 현재 전공과 학년은 무엇인가? | 전공, 학년 |
| 지금 가장 관심 있는 산업 또는 직무가 있는가? | 관심 산업, 관심 직무 |
| 취업/면접/포트폴리오 준비 기간은 어느 정도인가? | 2주, 4주, 8주, 12주 |
| 최근 2주 안에 찾아본 교육 자료가 있는가? | 자료명, 찾은 이유 |
| 직무를 고를 때 가장 불안한 점은 무엇인가? | 직무 이해, 역량 부족, 교육 선택, 산출물 부족 |

## 과업 시나리오

진행자는 아래 순서대로 과업만 제시하고, 학생이 화면을 조작하게 한다.

### 1. 전공과 준비 조건 선택

과업:

> 본인 전공과 가장 가까운 값을 선택하고, 관심 산업과 준비 기간을 고르세요.

관찰:

- 전공 선택지가 자기 배경을 충분히 담는다고 느끼는가?
- 산업 필터가 직무를 찾는 데 도움이 되는가?
- 준비 기간이 이후 추천 난이도나 분량에 영향을 준다고 예상하는가?

통과 신호:

- 학생이 전공을 "내 출발점"으로 이해한다.
- 산업 필터와 전공 선택을 같은 개념으로 혼동하지 않는다.

실패 신호:

- 전공이 직무명처럼 보인다고 말한다.
- 산업을 바꾸면 전공 적합도 자체가 바뀐다고 오해한다.

### 2. 상세 직무 카드 선택

과업:

> 추천된 직무 카드 중 하나를 고르고, 왜 이 직무를 선택했는지 말해주세요.

관찰:

- 학생이 카드 제목만 보고 고르는가, 상세 업무/산출물까지 읽는가?
- `전공 직결`, `전공 확장`, `도전 직무` 라벨을 이해하는가?
- 상세직무내용이 실제 업무로 읽히는가, 교육 과정명처럼 읽히는가?

통과 신호:

- 학생이 "이 직무는 이런 업무를 하고, 내 전공과는 이런 관계"라고 설명한다.
- 직무 카드의 산출물 또는 키워드를 근거로 선택한다.

실패 신호:

- 상세 직무를 전공 카테고리 또는 교육 과정으로 오해한다.
- `도전 직무`를 추천 불가 또는 부적합이라는 의미로 받아들인다.

### 3. 전공-상세직무-교육 구분 설명

과업:

> 지금 화면에서 전공, 상세직무내용, 직무 관련 교육이 각각 무엇을 의미하는지 본인 말로 설명해 주세요.

판정 기준:

| 항목 | 통과 기준 |
|---|---|
| 전공 | 학생 배경 또는 출발 지식으로 설명한다. |
| 상세직무내용 | 회사에서 수행할 업무, 문제, 산출물로 설명한다. |
| 직무 관련 교육 | 부족한 역량을 보완하는 학습 자료로 설명한다. |

중요 기록:

- 세 개념 중 어느 둘이 가장 자주 섞이는지 적는다.
- 용어가 어려워서 막히는지, UI 배치가 헷갈려서 막히는지 구분한다.

### 4. 역량 체크

과업:

> 본인이 이미 할 수 있거나 설명할 수 있는 항목을 체크하세요.

관찰:

- 체크 항목이 해당 직무의 실제 역량처럼 보이는가?
- 너무 추상적이거나 전공 수업명처럼 보이는 항목이 있는가?
- 전체 체크 후 나타나는 문구가 학생의 실제 자신감과 맞는가?

통과 신호:

- 학생이 체크한 역량과 추천 교육의 변화가 연결된다고 이해한다.
- 미체크 역량이 "지금 공부할 우선순위"로 받아들여진다.

실패 신호:

- 모든 항목을 체크했는데도 추천 교육 이유를 납득하지 못한다.
- `큰 공백 없음`이 "더 할 필요 없음"으로 읽힌다.

### 5. 교육 추천과 1-2주차 확인

과업:

> 첫 번째 추천 교육과 1-2주차 교육을 보고, 왜 이 순서인지 설명해 주세요.

관찰:

- 첫 번째 추천 교육이 직무 수행에 직접 도움이 된다고 느끼는가?
- 3시간 안팎의 짧은 교육이 여러 주차에 반복되어 보이는가?
- 기초 교육, 실습 교육, 심화 교육의 선후관계가 자연스러운가?
- Coursera, MathWorks Onramp, KOCW, 공식 문서 등 자료 출처가 신뢰 가능하게 보이는가?
- 한국어 가능 자료가 학생에게 실제 접근성 개선으로 느껴지는가?

통과 신호:

- 학생이 첫 추천 교육을 "왜 지금 먼저 해야 하는지" 설명한다.
- 1주차와 2주차의 역할 차이를 구분한다.
- 중복 자료가 반복 추천처럼 보이지 않는다.

실패 신호:

- 같은 자료가 반복되어 일정이 채워진 것처럼 보인다고 말한다.
- 심화 자료가 기초 자료보다 먼저 나온다고 느낀다.
- Coursera 자료가 좋아 보이지만 직무와 왜 연결되는지 모르겠다고 말한다.

### 6. 페르소나 문구 반응

과업:

> 화면의 학생 페르소나 설명이 본인 상황을 맞게 표현하는지 말해주세요.

우선 확인할 고위험 화면:

| QA 시나리오 | 확인 이유 |
|---|---|
| 단기 준비 학생 | 2주 일정에서 추천이 과밀하거나 중복되어 보일 위험 |
| 전공 확장 학생 | 전공과 직무의 연결 강도를 과장할 위험 |
| 도전 직무 학생 | 도전 직무가 부정적 평가처럼 읽힐 위험 |

관찰:

- 페르소나명이 학생을 낙인찍는 느낌을 주는가?
- 설명이 너무 일반적이라 추천 근거로 느껴지지 않는가?
- 추천 교육보다 페르소나 문구가 더 눈에 띄어 흐름을 방해하는가?

통과 신호:

- 학생이 "내 상황과 대체로 맞다" 또는 "이 부분만 다르다"고 구체적으로 반응한다.
- 페르소나 문구가 다음 행동 선택을 돕는다.

실패 신호:

- 학생이 왜 이 페르소나가 나왔는지 설명하지 못한다.
- 페르소나명이 부담스럽거나 부정확하다고 느낀다.

### 7. 내 커리큘럼 추가와 내보내기

과업:

> 실제로 도움이 될 것 같은 교육을 내 커리큘럼에 추가하고, 결과물을 확인하세요.

관찰:

- 학생이 어떤 기준으로 교육을 저장하는가?
- 저장한 교육이 최종 로드맵에서 중복되거나 의미 없이 반복되는가?
- 엑셀/커리큘럼 결과물이 상담, 면접 준비, 학습 계획에 쓸 수 있어 보이는가?

통과 신호:

- 학생이 최소 1개 교육을 자기 계획에 추가한다.
- 최종 산출물이 다음 학습 행동으로 이어진다.

실패 신호:

- 추천은 읽었지만 저장할 만큼 유용하지 않다고 판단한다.
- 최종 결과물이 화면 요약과 다르지 않아 내보내기 필요성을 느끼지 못한다.

## 페르소나 매핑 기록

인터뷰 후 학생을 아래 7개 자동 QA 페르소나 중 가장 가까운 하나에 매핑한다. 앱의 자동 판정과 학생 자기인식이 다르면 그 차이를 기록한다.

| QA 페르소나 | 인터뷰에서 볼 신호 |
|---|---|
| 직무 미확신/역량 체크 전 학생 | 직무 선택 전에 오래 머물고 역량 체크를 망설임 |
| 단기 준비 학생 | 2주 안에 바로 할 자료와 산출물을 요구함 |
| 전공 확장 학생 | 전공은 비슷하지만 산업/도메인 용어가 낯섦 |
| 도전 직무 학생 | 전공과 거리감이 크고 기초 학습 순서를 중요하게 봄 |
| 기초 보완 학생 | 미체크 역량이 많고 첫 학습 자료 난이도에 민감함 |
| 실행 계획 학생 | 이미 저장한 교육이 있고 구체적인 일정화를 원함 |
| 전공 직결 학생 | 직무는 이해하지만 포트폴리오 산출물 구체화를 원함 |

## 합격/보류/수정 판정

각 인터뷰는 아래 기준으로 종료 판정한다.

| 판정 | 기준 |
|---|---|
| 합격 | 학생이 전공, 상세직무내용, 직무 관련 교육을 구분하고 첫 추천 교육의 이유를 설명한다. |
| 보류 | 큰 오해는 없지만 특정 용어, 문장, 추천 순서에 망설임이 있다. |
| 수정 필요 | 직무와 교육을 혼동하거나, 첫 추천 교육이 직무에 도움이 된다고 납득하지 못한다. |

수정 필요가 2명 이상 반복되면 코드 수정 우선순위로 올린다. 단, 개별 학생의 취향 문제인지 구조적 오해인지 구분한다.

## 인터뷰 후 반영 우선순위

1. 첫 추천 교육이 직무와 왜 연결되는지 설명하지 못한 사례
2. 전공, 상세직무내용, 교육을 혼동한 사례
3. 1-2주차 중복 또는 선후관계 역전으로 보인 사례
4. 페르소나 문구가 낙인, 과장, 과도한 낙관으로 읽힌 사례
