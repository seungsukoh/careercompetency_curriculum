## 완료한 변경

### 1. 공고와 오늘 시작할 1개 연결 개선

`오늘 시작할 1개`가 더 이상 주차 과제 추천 순서를 그대로 따르지 않는다. 이제 아래 신호를 함께 본다.

- 현재 미체크 역량
- 이미 체크한 역량
- 공고 문장과 교육자료의 겹침
- 선택 직무와 직접 연결된 자료인지
- 이미 완료한 자료인지

핵심 QA 케이스:

| 케이스 | 체크 상태 | 오늘 시작할 자료 |
| --- | --- | --- |
| 생산 데이터 분석 공고 | 기초 미체크 | `nist-control-charts` |
| 생산 데이터 분석 공고 - 기초 체크 완료 | 통계, SPC, 데이터 분석, 데이터 전처리, MES/SQL, Pareto 체크 | `machine-learning-onramp` |

PM 판단: 같은 공고라도 학생 준비 상태에 따라 시작 자료가 달라지는 동작이 검증됐다.

### 2. 자료 신뢰도 기준 강화

`candidate` 또는 `reviewNeeded` 자료는 데이터에 `core: true`가 있어도 앱 내부에서 필수/핵심 자료로 취급하지 않도록 막았다.

변경된 기준:

- `verified`, `reviewed`: 필수 Starter Pack 또는 핵심 교육 가능
- `candidate`, `reviewNeeded`: 추천 후보로는 가능하지만 필수/핵심 라벨과 보너스 제외

조정한 자료:

| 자료 | 이전 문제 | 현재 상태 |
| --- | --- | --- |
| `kosha-psm` | 기관 신뢰도는 높지만 링크가 KOSHA 메인이고 `candidate` 상태 | 필수/핵심 제외, 후보 자료 유지 |
| `lean-a3-problem-solving` | `candidate`인데 핵심 자료처럼 보일 수 있음 | 핵심 제외, 후보 자료 유지 |

PM 판단: 필수자료라고 말할 수 있는 기준이 더 보수적으로 정리됐다. KOSHA는 실제 단원 링크 확인 후 `reviewed` 승격 여부를 결정하는 것이 맞다.

### 3. 데이터 검증 강화

`scripts/validate-data.mjs`에 품질 상태 검증을 추가했다.

앞으로 아래 상태가 생기면 `validate:data`가 실패한다.

- 알 수 없는 `qualityStatus`
- `candidate` 또는 `reviewNeeded` 자료가 `core` 또는 `starterPack`으로 들어오는 경우
- 검토되지 않은 자료가 필수/핵심 후보 플래그를 요청하는 경우

PM 판단: 자료 큐레이션 기준이 코드 레벨에서 회귀 방지된다.

## 현재 남은 리스크

- 아직 커밋/푸시되지 않았다.
- Pages 배포본에서 최신 JS asset이 반영됐는지 확인해야 한다.
- `KOSHA PSM`은 기관 신뢰도는 높지만 현재 앱 데이터는 KOSHA 메인 URL이므로 필수자료 승격 전 실제 문서/교육 단원 링크 확인이 필요하다.
- `오늘 시작할 1개`의 내부 추천 근거는 개선됐지만, 사용자가 화면에서 왜 그 자료가 골라졌는지 보는 근거 표시는 아직 충분하지 않다.

## 다음 추천 액션

1. 현재 변경을 커밋하고 푸시한다.
2. Pages 배포 후 실제 화면에서 공고 대조와 오늘 시작할 자료를 확인한다.
3. `오늘 시작할 1개` 카드에 추천 근거를 더 명확히 표시한다.
   - 예: 공고 키워드, 보완 역량, 이미 체크한 기초 제외, 직무 직접 연결
4. KOSHA, A3 등 후보 자료의 실제 단원 링크를 검수해 `reviewed` 승격 여부를 결정한다.
5. PM/교육자료 오너가 볼 수 있는 자료 신뢰도 목록을 별도 표로 만든다.
