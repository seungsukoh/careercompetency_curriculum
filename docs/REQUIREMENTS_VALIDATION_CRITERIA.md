# 요구사항 기반 검증 기준

## 목적

이 문서는 기능, 자료, UI, 운영 절차를 현재 제품 방향에 맞춰 검증하기 위한 기준이다. 기준 흐름은 `직무 선택 -> 워드클라우드 -> 역량 체크 -> 기간 맞춤 자동 로드맵 -> 내 커리큘럼`이다.

## 제품 범위 기준

1차 제품은 공학 전공 취준생이 혼자 사용할 `직무 기반 자동 로드맵 도구`다. 현재 구현 기준은 16개 직무군, 96개 세부 직무, 123개 교육자료다.

포함한다:

- 학생이 전공, 관심 산업, 학습목표, 준비 기간을 고른다.
- 앱이 채용공고 수준으로 세분화된 직무를 추천 순서로 보여준다.
- 직무 선택 즉시 역량 워드클라우드를 보여준다.
- 역량 체크는 선택 직무와 관심 산업에 따라 달라진다.
- 로드맵은 교육자료를 먼저 고르지 않아도 기간에 맞춰 자동 구성된다.
- 실제 교육자료는 로드맵 주차 안에서 확인하고 필요하면 내 커리큘럼에 추가한다.
- 내 커리큘럼은 완성된 로드맵을 보여주고 엑셀로 내보낼 수 있다.
- 참고자료 보관함은 전체 자료를 확인하는 보조 화면이며, 핵심 교육 선택 흐름은 아니다.

지금은 포함하지 않는다:

- 로그인, 결제, 유료 기능
- 교수자 관리자 화면
- 전체 웹 자동 크롤링
- AI가 검수 없이 자료를 생성하거나 추천하는 기능
- 학생 개인정보와 산출물 파일 수집
- 학년 기반 추천

## 요구사항 목록

| ID | 요구사항 | 우선순위 | 검증 질문 |
|---|---|---|---|
| R1 | 학생은 채용공고 수준으로 세분화된 직무를 선택할 수 있어야 한다. | P0 | 학생이 직무명과 반복 키워드로 지원 직무를 고를 수 있는가 |
| R2 | 직무 선택 즉시 워드클라우드가 보여야 한다. | P0 | 선택 직무의 반복 역량이 바로 보이는가 |
| R3 | 직무 상세는 실제 업무, 자격요건, 우대 역량, 확인 검색어를 포함해야 한다. | P0 | 직무명을 넘어 무엇을 준비해야 하는지 설명할 수 있는가 |
| R4 | 진단은 선택 직무와 관심 산업에 따라 달라져야 한다. | P0 | 같은 트랙이라도 선택 직무가 바뀌면 체크 항목이 달라지는가 |
| R5 | 학습목표는 로드맵 과제 우선순위와 교육자료 점수에 영향을 줘야 한다. | P0 | 직무 탐색/전공 보완/포트폴리오/면접 준비 선택에 따라 기준이 바뀌는가 |
| R6 | 준비 기간은 로드맵 주차 수, 반복 과제, 교육자료 수를 바꿔야 한다. | P0 | 2/4/8/12주 선택이 결과에 실질적으로 반영되는가 |
| R7 | 교육자료는 자료 탭이 아니라 로드맵 주차 안에서 확인되어야 한다. | P0 | 주차별 과제와 연결된 자료를 바로 열 수 있는가 |
| R8 | 자료 추천은 직무, 진단 공백, 채용 키워드, 목표, 기간, MathWorks 필요성을 함께 계산해야 한다. | P0 | 자동 배치 근거가 납득되는가 |
| R9 | 내 커리큘럼은 완성된 로드맵 화면이어야 한다. | P0 | 교육 선택 화면과 달리 실행/완료/내보내기 용도로 보이는가 |
| R10 | 내 커리큘럼은 엑셀 형태로 내보낼 수 있어야 한다. | P0 | 상담이나 개인 계획 관리에 쓸 수 있는 파일이 생성되는가 |
| R11 | 자료, 진단 항목, 워드클라우드 용어는 월 1회 또는 학기 시작 전 검수로 갱신 가능해야 한다. | P0 | 운영자가 매주 수동 검색하지 않아도 유지 가능한가 |
| R12 | MVP는 정적 PWA로 유지해야 한다. | P0 | 백엔드 없이 배포와 사용이 가능한가 |
| R13 | 저작권과 출처 리스크를 줄여야 한다. | P0 | 원문 복제 없이 제목, 요약, 추천 이유, 원본 링크 중심인가 |

## 인수 기준

| 영역 | 통과 기준 |
|---|---|
| 직무 선택 | 16개 직무군과 96개 세부 직무가 보이고, 선택 즉시 워드클라우드가 표시된다. |
| 역량 체크 | 트랙 공통, 선택 직무, 관심 산업 기준 체크 항목이 함께 반영된다. |
| 로드맵 구성 | 자료 선행 선택 없이 기간과 목표에 맞춘 주차별 과제가 생성된다. |
| 교육자료 연결 | 각 주차에 2~3개 교육자료가 연결되고 점수와 자동 배치 근거가 보인다. |
| MathWorks | CAE, 제어, 품질/수율, 전원/모터 등 필요한 직무에서 MathWorks 자료가 상위에 반영된다. |
| 내 커리큘럼 | 완성 로드맵, 주차 완료 체크, 추가 자료, 엑셀 내보내기가 동작한다. |
| 모바일 | 직무 카드, 동적 워드클라우드, 주차 자료, 내보내기 버튼이 겹치지 않는다. |
| 운영 | 자료 검수상태, 확인일, 직무 키워드와 워드클라우드 용어 갱신 절차가 유지된다. |

## 검증 방법

| 검증 유형 | 대상 | 방법 | 산출물 |
|---|---|---|---|
| 요구사항 리뷰 | PM/교수자/아키텍처 | 요구사항이 현재 흐름과 맞는지 확인 | 이 문서의 요구사항 수정 |
| 데이터 감사 | 개발/큐레이션 | `npm.cmd run validate:data`로 role/resource/task 링크 누락 검사 | 감사 결과 |
| 기능 QA | 앱 UI/UX | 로컬 브라우저에서 핵심 플로우 수행 | `docs/QA_CHECKLIST.md` 결과 |
| 모바일 QA | 앱 UI/UX | 휴대폰 폭에서 선택, 진단, 로드맵, 내보내기 확인 | 스크린샷 또는 체크 결과 |
| 운영성 리뷰 | PM/아키텍처 | 월간 자료 갱신 절차 확인 | 운영 부담 항목과 보류 결정 |

## 전문가 그룹 검토 관점

| 관점 | 검토 책임 |
|---|---|
| PM | 범위, 우선순위, 성공 기준, 보류 기능 결정 |
| 진로/교수자 | 취준생이 선택 기준을 이해하는지, 상담에 쓸 수 있는지 검토 |
| 산업 도메인 | 직무 설명, 진단 항목, 역량 연결이 현실적인지 검토 |
| 자료 큐레이션 | 자료 품질, 최신성, 주차 산출물 연결성 검토 |
| SW 아키텍처 | 정적 PWA, 데이터 경계, 마이그레이션 가능성 검토 |
| QA | 요구사항별 통과/실패를 재현 가능하게 기록 |

## 다음 작업 판단

1. P0 흐름은 `docs/QA_CHECKLIST.md`와 최신 `docs/QA_RESULTS_*.md`로 검증한다.
2. 자료 후보는 로드맵 과제와 직접 연결되는 경우에만 추가한다.
3. 워드클라우드는 검수된 직무/진단/자료 데이터에서 동적으로 렌더링한다. 정적 PNG를 다시 쓰는 경우에만 MATLAB 생성 절차를 사용한다.
4. P1/P2 기능은 학생 파일럿 결과가 나온 뒤 결정한다.
