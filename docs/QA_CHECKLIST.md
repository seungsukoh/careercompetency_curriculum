# QA 체크리스트

## 사용 원칙

이 체크리스트는 `docs/REQUIREMENTS_VALIDATION_CRITERIA.md`의 요구사항을 검증하기 위한 실행 목록이다. QA는 새 기능을 제안하기보다 요구사항별 통과/실패와 재현 절차를 남긴다.

검증 환경:

- 로컬 서버: `python -m http.server 8787`
- URL: `http://localhost:8787`
- 브라우저: Chrome 또는 Edge 최신 버전
- 화면 폭: 데스크톱 1280px 이상, 모바일 390px 안팎

## 기능 검증

| ID | 요구사항 | 확인 절차 | 기대 결과 | 결과 |
|---|---|---|---|---|
| R1 | 직무 트랙 선택 | 첫 화면에서 필터를 바꾸고 직무 트랙을 선택한다. | 선택한 트랙이 강조되고 상세/진단/로드맵/자료가 같은 트랙 기준으로 바뀐다. | 미검증 |
| R2 | 직무 이해 | 직무 탭에서 선택 트랙의 설명을 확인한다. | 실제 업무, 필요 역량, 도구, 산출물, 흔한 오해가 보인다. | 미검증 |
| R3 | 역량 진단 | 진단 탭에서 체크박스를 여러 개 선택/해제한다. | 점수와 부족 역량 목록이 즉시 갱신된다. | 미검증 |
| R4 | 학습 순서 | 자료 탭에서 추천 자료 목록을 확인한다. | 자료가 추천 순서와 난이도 기준으로 정렬되어 있다. | 미검증 |
| R5 | 자료 메타데이터 | 자료 카드 하나를 확인한다. | 난이도, 언어, 예상 시간, 선행조건, 산출물, 검수상태가 표시된다. | 미검증 |
| R6 | 한국어 우선 | 자료 목록의 상단 자료들을 확인한다. | 한국어 접근 가능 자료가 우선 노출된다. | 미검증 |
| R8 | 자료 저장/완료 | 자료를 저장하고 완료 체크한 뒤 새로고침한다. | 저장/완료 상태가 유지된다. | 미검증 |
| R9 | 로드맵 완료 | 로드맵 탭에서 주차 완료를 체크한 뒤 새로고침한다. | 완료 상태가 유지된다. | 미검증 |
| R10 | 결과 복사 | 저장/완료 후 파일럿 결과 복사를 실행한다. | 복사된 요약에 선택 트랙, 진단, 저장/완료 자료, 완료 주차가 포함된다. | 미검증 |
| R13 | 정적 PWA | 로컬 서버에서 새로고침하고 주요 탭을 이동한다. | 서버 API 없이 앱이 동작한다. | 미검증 |

## 모바일 검증

| 항목 | 확인 절차 | 기대 결과 | 결과 |
|---|---|---|---|
| 첫 화면 | 모바일 폭에서 앱을 연다. | 핵심 CTA와 직무 선택이 보이고 가로 스크롤이 없다. | 미검증 |
| 탭 이동 | 직무, 진단, 로드맵, 자료, 저장 탭을 누른다. | 버튼 텍스트가 겹치지 않고 탭 전환이 가능하다. | 미검증 |
| 자료 카드 | 자료 탭에서 카드 내용을 읽는다. | 긴 제목과 메타데이터가 카드 밖으로 넘치지 않는다. | 미검증 |
| 완료 체크 | 자료 완료와 로드맵 완료를 누른다. | 터치 대상이 충분하고 상태 변화가 명확하다. | 미검증 |
| 복사 | 파일럿 결과 복사를 누른다. | 모바일 브라우저에서도 복사 또는 대체 안내가 동작한다. | 미검증 |

## QA 기록 양식

검증 후 아래 형식으로 기록한다.

```text
검증일:
검증자:
브라우저/화면:

통과:
- R1:
- R2:

실패:
- 요구사항 ID:
- 재현 절차:
- 기대 결과:
- 실제 결과:
- 심각도: P0/P1/P2

보류:
- 사유:
```

## 수정 우선순위

1. P0 요구사항 실패는 파일럿 전에 수정한다.
2. 모바일에서 핵심 흐름을 막는 문제는 P0로 본다.
3. 자료 품질 이슈는 잘못된 학습 순서를 만들면 P0, 보완 가능한 설명 부족이면 P1로 본다.
4. 운영자가 반복 노동을 하게 만드는 변경은 기능을 줄이거나 보류한다.
