# PM 실행 계획

## 현재 PM 결론

1차 제품은 학생이 자료를 직접 고르는 앱이 아니라, 관심 직무와 준비 기간을 기준으로 검수된 교육자료를 주차별 로드맵에 자동 배치하는 도구다.

현재 구현 기준은 `16개 직무군`, `96개 세부 직무`, `127개 교육자료`다. 직무군 수는 초기 파일럿 최소 범위를 넘어섰으므로, 새 기능 확장보다 검증과 배포 안정화를 우선한다.

핵심 약속:

> 직무를 고르면 바로 역량 워드클라우드를 보고, 역량 체크 상태와 준비 기간에 맞춘 자동 로드맵 안에서 필요한 교육자료를 확인한다.

## 1차 MVP 사용자

주 사용자:

- 기계공학, 전자공학, 화학공학 전공 취준생
- 혼자 직무 준비를 시작하려는 학생
- 무엇을 먼저 공부해야 할지 모르는 학생

보조 사용자:

- 상담 전에 학생의 관심 직무와 준비 상태를 보고 싶은 교수자/취업지원 담당자

## 해결할 문제

- 직무명이 너무 넓어 실제 채용공고 수준의 준비 항목을 알기 어렵다.
- 공개 교육자료는 많지만 어떤 자료를 어느 주차에 써야 할지 판단하기 어렵다.
- 학습목표와 준비 기간이 달라도 같은 추천을 받으면 실행성이 떨어진다.
- 공부한 결과가 포트폴리오나 면접 설명 근거로 남지 않는다.

## MVP 범위

포함:

- 16개 직무군과 96개 세부 직무
- 직무 선택 즉시 워드클라우드 표시
- 선택 직무와 관심 산업 기반 역량 체크
- 2/4/8/12주 기간 맞춤 자동 로드맵
- 주차별 추천 교육자료, 자동추천 점수, 배치 근거, 원본 링크
- MathWorks, Coursera, K-MOOC/KOCW/HRD-Net/GSEEK 등 공개자료 매핑
- 내 커리큘럼에서 완성 로드맵 확인, 주차 완료 체크, 엑셀 내보내기
- 참고자료 보관함에서 전체 자료 확인

제외:

- 자료 탐색을 핵심 흐름으로 만드는 별도 자료 선택 탭
- 학년 기반 추천
- 회원가입/로그인
- 결제/유료 기능
- 교수자 관리자 페이지
- 전체 웹 자동 크롤링
- AI가 검수 없이 교육자료를 생성하는 기능

## 성공 기준

학생 5명 파일럿에서 다음을 확인한다.

| 지표 | 목표 |
|---|---|
| 세부 직무 선택 완료 | 5명 중 4명 이상 |
| 워드클라우드가 직무 이해에 도움이 됐다는 응답 | 5명 중 3명 이상 |
| 역량 체크 완료 | 5명 중 3명 이상 |
| 자동 로드맵이 실행 가능하다는 응답 | 5명 중 3명 이상 |
| 주차별 교육자료가 과제와 연결된다는 응답 | 5명 중 3명 이상 |
| 내 커리큘럼 또는 엑셀 내보내기 사용 | 5명 중 2명 이상 |

정성 질문:

- 선택한 직무명이 채용공고에서 본 수준과 가까운가
- 첫 주에 무엇을 해야 하는지 명확한가
- 추천 교육자료가 너무 넓거나 뜬금없지 않은가
- 기간을 바꿨을 때 로드맵이 현실적으로 달라지는가
- 빠진 직무, 역량, 교육자료가 무엇인가

## 우선순위

### P0: 파일럿 전에 반드시 필요

1. 세부 직무 선택과 직무별 워드클라우드
2. 선택 직무/산업 기반 진단
3. 기간 맞춤 자동 로드맵
4. 주차별 교육자료 연결과 자동 배치 근거
5. MathWorks 필요 직무의 자료 매핑
6. 내 커리큘럼 엑셀 내보내기
7. 모바일 핵심 흐름 검증

### P1: 파일럿 중 개선

1. 직무별 자료 후보 추가 검수
2. Coursera/edX/K-MOOC/HRD-Net 후보의 실제 강좌 단원 지정
3. 학생 피드백 기반 로드맵 과제 문장 보정
4. 직무군별 워드클라우드 시각 품질 개선

### P2: 파일럿 이후

1. `app.js` seed 데이터를 `data/*.json`으로 분리
2. 운영자용 자료 검수 화면
3. 실제 채용공고 수집 자동화
4. 계정 기반 저장

## 이번 작업의 PM 결정

- 자료 선택 전용 탭은 만들지 않고, 교육자료는 로드맵 주차 안에서 확인한다.
- 참고자료는 보조 보관함이고, 교육 선택의 핵심은 로드맵 주차 안에 둔다.
- 내 커리큘럼은 북마크 목록이 아니라 완성된 로드맵과 내보내기 화면이다.
- 학년은 추천 기준에서 제외한다.
- 학습목표는 과제 우선순위와 교육자료 점수에 모두 반영한다.
- 운영자는 월 1회 검수 자료, 직무 키워드, 진단 항목을 반영한다. 현재 앱의 워드클라우드는 seed 데이터 기반 동적 렌더링이므로 PNG 재생성은 정적 이미지 사용 시에만 필요하다.
