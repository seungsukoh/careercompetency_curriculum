# PM 실행 계획

## 현재 PM 결론

1차 제품은 학생이 혼자 공부할 때 사용할 자기주도 학습 자료 큐레이션 도구다. 교수자/관리자 플랫폼, AI 자동 생성기, 전체 취업 준비 서비스로 확장하지 않는다.

개발 진행 속도보다 방향성과 유지보수성을 우선한다. 새 기능을 추가하기 전에 데이터 구조, 운영 부담, 향후 마이그레이션 경계를 먼저 확인한다.

핵심 약속:

> 관심 직무를 고르면, 검수된 한국어 자료를 학습 순서대로 배치하고, 예상 소요시간과 산출물을 보여준다.

## 1차 MVP 사용자

주 사용자:

- 기계공학/전자공학 2~4학년
- 혼자 취업/직무 준비를 시작하려는 학생
- 무엇을 먼저 공부해야 할지 모르는 학생

보조 사용자:

- 상담 전에 학생의 관심 직무와 준비 상태를 보고 싶은 교수자/취업지원 담당자

## 해결할 문제

학생이 직접 느끼는 문제:

- 자료가 너무 많아서 고르기 어렵다.
- 유튜브/강의는 많은데 어떤 순서로 봐야 할지 모른다.
- 내 수준에 맞는지 판단하기 어렵다.
- 공부해도 포트폴리오나 면접 소재로 남지 않는다.

학생이 잘 모를 수 있는 문제:

- 직무별로 필요한 전공 기초가 다르다.
- 인기 자료가 항상 좋은 학습 경로는 아니다.
- 자료는 시청보다 산출물로 연결되어야 한다.

## MVP 범위

포함:

- 직무 트랙 5개
- 직무별 실제 업무, 필요 역량, 산출물 설명
- 역량 체크리스트
- 한국어 우선 자료 추천
- 자료별 추천 순서, 난이도, 예상 소요시간, 선행조건, 산출물
- 자료 저장/완료 체크
- 로드맵 주차별 수동 완료 체크
- 파일럿 결과 복사

제외:

- 회원가입/로그인
- 결제/유료 기능
- 교수자 관리자 페이지
- AI 자동 커리큘럼 생성
- 기업별 상세 직무 매핑
- 전체 자료 자동 크롤링

## 성공 기준

학생 5명 파일럿에서 다음을 확인한다.

| 지표 | 목표 |
|---|---|
| 직무 트랙 선택 완료 | 5명 중 4명 이상 |
| 역량 체크리스트 완료 | 5명 중 3명 이상 |
| 추천 자료 저장 | 5명 중 3명 이상 |
| 완료 체크 1개 이상 | 5명 중 2명 이상 |
| "왜 이 순서인지 이해됨" 응답 | 5명 중 3명 이상 |

정성 질문:

- 이 자료 묶음이 직접 검색보다 나았는가
- 첫 번째로 볼 자료가 명확했는가
- 예상 소요시간이 학습 계획에 도움이 되었는가
- 산출물이 있으면 공부할 이유가 더 분명해졌는가
- 빠진 직무/자료/설명이 무엇인가

## 우선순위

### P0: 파일럿 전에 반드시 필요

1. 자료 카드에 추천 순서, 시간, 산출물 표시
2. 자료 완료 체크와 로드맵 완료 체크
3. 모바일에서 핵심 흐름 검증
4. 한국어 자료 후보를 트랙별 최소 3개씩 확보
5. 파일럿 결과 복사 내용에 완료 상태 포함

### P1: 파일럿 중 개선

1. 트랙별 한국어 자료를 5개 이상으로 확대
2. 자료 품질 상태 `candidate/reviewed/verified` 정리
3. 학생 피드백 수집 문항 보강
4. README 실행/배포 절차 정리

### P2: 파일럿 이후

1. `app.js` seed 데이터를 `data/*.json`으로 분리
2. 학생 상황별 1주/2주/4주 계획 조절
3. 교수자 상담용 요약 강화
4. 링크/조회수/댓글 상태 자동 점검 검토

## 병렬 작업 지시

현재 병렬 흐름은 `docs/WORKSTREAMS.md`를 따른다.

PM은 다음 결정을 관리한다.

- MVP 범위 변경 승인
- 자료 품질 기준 승인
- 파일럿 성공 기준 승인
- 유료화/계정/교수자 기능의 도입 시점 보류 또는 승인

개발자는 다음을 맡는다.

- 정적 PWA 구현
- 자료 카드/완료 체크 UI
- localStorage 상태 저장
- 브라우저 검증

큐레이션 담당은 다음을 맡는다.

- 한국어 자료 후보 수집
- 조회수/댓글/공식성/최신성 확인
- 추천 순서와 산출물 초안 작성

## 이번 작업의 PM 결정

- 1차 제품은 "자료를 찾는 앱"이 아니라 "검수된 자료를 순서대로 배치해주는 앱"이다.
- 운영자가 매주 관리해야 하는 구조는 피한다.
- 트랙별 작고 안정적인 자료 묶음으로 시작한다.
- 학생 동기부여를 위해 저장뿐 아니라 완료 체크를 제공한다.
- SW 아키텍처는 나중에 DB/유료화로 옮길 수 있도록 데이터 필드를 유지한다.
- 유지보수성을 해치는 기능 추가는 보류한다.
- 아키텍처 결정은 `docs/ARCHITECTURE_DECISIONS.md`에 기록한다.

## 다음 PM 체크포인트

1. 트랙별 한국어 자료 후보가 충분한가
2. 자료 순서가 학생 눈에 납득되는가
3. 소요시간이 현실적인가
4. 산출물이 너무 무겁거나 모호하지 않은가
5. 학생 5명에게 보여줄 링크 수준까지 앱이 안정적인가
